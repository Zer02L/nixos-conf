# AI Agent Built-in Tools

Инструменты, которыми агент пользуется для работы с файлами и кодом. Все — локальные, без обращений к облаку.

## Поиск и чтение

| Команда | Что делает | Вместо... |
|---|---|---|
| `read <path>` | Читает файл/директорию/архив/URL/SQLite/изображение | `cat`, `ls`, `head`, `curl`, `unzip`, `file` |
| `search "<regex>" [paths]` | Regex-поиск по содержимому файлов | `grep`, `rg`, `git grep` |
| `find [paths]` | Поиск по имени/глобу | `fd`, `find -name` |
| `lsp definition/references/..` | LSP-запросы (типы, референсы, rename) | grep по импортам / sed rename |

## Редактирование

| Команда | Что делает | Вместо... |
|---|---|---|
| `edit` | Строчные правки с line range + tag | `sed -i`, `awk -i` (безопаснее — знает структуру строк) |
| `write <path>` | Создать/перезаписать файл | `>`, `echo >`, heredoc |
| `ast_grep` | Поиск по AST (структура кода, не текст) | grep не понимает синтаксис |
| `ast_edit` | Структурная замена по AST | `sed` вслепую (замена только настоящих обращений) |

## Исполнение и вычисления

| Команда | Что делает | Вместо... |
|---|---|---|
| `eval` | Python / JS kernel (интерактивно) | `python -c`, `node -e` |
| `bash` | Любая команда (последнее средство) | fallback, если нет dedicated tool |
| `browser` | Chromium с executeJS + observe | SPA не тянутся через curl |
| `debug` | DAP (launch/breakpoints/step) | `gdb`, `pdb` |

## Оркестрация

| Команда | Что делает |
|---|---|
| `task` | Параллельные subagent-ы для независимых задач |
| `todo` | Управление списком подзадач внутри сессии |

## Оптимизация

- **Вход**: компактный JSON, не shell pipeline с флагами
- **Выход**: схлопнутые тела функций (`..`/`…`), `*` на строке совпадения, обрезанный >50KB в `artifact://`
- **Семантика**: `ast_edit` меняет только настоящие обращения к символу, не трогает строки с совпадением по тексту
- **offline**: ни один инструмент не требует сети
